## 02. 리스트

* 리스트는 하나의 변수에 여러 값을 할당하는 자료형이다. 이러한 기법을 시퀀스 자료형이라고 한다. 리스트는 하나의 자료형으로만 저장하지 않고, 정수형이나 실수형 같은 다양한 자료형을 포함할 수 있다



### 2.1 리스트



### (예시)

* 리스트는 대괄호를 사용하여 입력한다

```python
cities = ['서울', '부산', '인천', '대구', '대전', '광주', '울산', '수원']
```



### 2.2 인덱싱과 슬라이싱



### (예시1)

* 인덱싱

* 인덱싱이란 리스트에 값에 접근하기 위해, 값의 상대적인 주소를 사용하는 것이다
* 인덱스는 0부터 시작한다

```python
cities[2]
```

```
'인천'
```



<img src="image/3/3-1.png" alt="3-1" style="zoom:80%;" />



### (예시2)

* 슬라이싱

* 슬라이싱이란 리스트의 잘라내어 반환하는 것이다
* 변수명[시작 인덱스: 마지막 인덱스]
* 마지막 인덱스는 포함하지 않는다

```python
cities[0:5]
```

```
['서울', '부산', '인천', '대구', '대전']
```



```python
cities[5:]
```

```
['광주', '울산', '수원']
```



### (예시3)

* 리버스 인덱싱

* 리버스 인덱싱은 기존 인덱스와 달리 마지막 값부터 -1을 할당하여 첫 번째 값까지 역순으로 올라오는 방식이다

```python
cities[-8:]
```

```
['서울', '부산', '인천', '대구', '대전', '광주', '울산', '수원']
```



<img src="image/3/3-2.png" alt="3-2" style="zoom:80%;" />



### (예시4)

* 슬라이싱을 할때 인덱스의 첫 번째 값이나 마지막 값이 비어 있어도 잘 작동한다
* 인덱스 범위를 넘어가는 슬라이싱

```python
# 다음 두 코드는 결과가 동일하다
cities[:]
cities[-50:50]
```

```
['서울', '부산', '인천', '대구', '대전', '광주', '울산', '수원']
```

### (예시5)

* 대괄호에 3번째 인수는 증가값을 나타낸다

```

```

```python
['서울', '인천', '대전', '울산']
```



* -1을 넣으면 역순으로 슬라이싱한다

```python
cities[::-1]
```

```
['수원', '울산', '광주', '대전', '대구', '인천', '부산', '서울']
```

### 2.4 리스트의 추가 및 삭제



### (예시)

```python
color.append('white')
```



### (정리)

|  메서드  |                             설명                             |               예시                |
| :------: | :----------------------------------------------------------: | :-------------------------------: |
| append() |            새로운 값을 기존 리스트의 맨 끝에 추가            |       color.append('white')       |
| extend() |  새로운 리스트를 기존 리스트에 추가(덧셈 연산과 같은 효과)   | color.extend(['black', 'purple']) |
| insert() | 기존 리스트의 i번째 인덱스에 새로운 값을 추가, i번째 인덱스를 기준으로 뒤쪽의 인덱스는 하나씩 밀림 |     color.insert(0, 'orange')     |
| remove() |                  리스트 내의 특정 값을 삭제                  |       color.remove('white')       |
|   del    |                     특정 인덱스값을 삭제                     |           del color[0]            |



### 2.5 패킹과 언패킹

* 패킹과 언패킹은 리스트에서만 사용하는 개념은 아니다. 다른 시퀀스 자료형에서 일반적으로 사용할 수 있는 방법이다



### (예시)

* 패킹은 한 변수에 여러개의 데이터를 할당하는 것, 그 자체를 패킹이라고 하며, 리스트 자체를 뜻하기도 한다

```python
# 1, 2, 3 을 변수 t에 패킹
t = [1, 2, 3]
```



* 언패킹은 한 변수에 여러개의 데이터가 들어 있을 때, 그것을 각각의 변수로 반환하는 방법이다

```python
# t에 있는 값을 a, b, c에 언패킹
a, b, c = t
```

## 03. 리스트의 메모리 관리 방식



### 3.1 리스트의 메모리 저장

* 리스트는 값을 저장하는 것이 아닌 주소를 저장한다



### (예시1)

```python
a = [1, 2, 3]
b = [4, 5, 6]
c = [7, 8, 9]
abc = [a, b, c]

b[0] = 1000
abc
```

```
[[1, 2, 3], [1000, 5, 6], [7, 8, 9]]
```



<img src="image/3/3-3.png" alt="3-3" style="zoom:80%;" />



### (예시2)

* is 는 메모리 주소를 비교하는 연산
* == 는 값을 비교하는 연산
* 파이썬의 정수형 저장 방식의 특성은 -5 ~ 256까지의 정수는 특정 메모리 주소 값에 저장되기에 두번째 블록 a is b 는 True를 나타낸다

```python
a = 300
b = 300

a is b # False
a == b # True
```

```python
a = 1
b = 1

a is b # True 
a == b # True
```



### 3.2 메모리 저장 구조로 인한 특징



### (예시1)

* 파이썬의 리스트가 값이 아닌 메모리의 주소를 저장해 다양한 자료형을 포함할 수 있다

```python
a = [1, 'a', True]
```



### (예시2)

* b=a를 입력하는 순간 b 리스트도 a 리스트의 메모리 주소와 연결이 된다. 두 변수가 같은 메모리 주소와 연결되어 있으므로, 하나의 변수 값만 바뀌더라도 둘 다 영향을 받는 것이다.

```python
a = [5, 4, 3, 2, 1]
b = [1, 2, 3, 4, 5]

b = a
a.sort()
b
```

```
[1, 2, 3, 4, 5]
```
